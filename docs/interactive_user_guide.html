SCANLIGHT OPERATOR GUIDE

Film Scanner

Interactive guide for the Swift app, Sony Wi-Fi SDK path, manual IED fallback, calibration, and current limitations.

Current rig assumption: Sony a7CR over Wi-Fi PC Remote, Scanlight over USB serial, Valoi film advance by hand.

WHAT THIS APP IS FOR

Make a camera scanning rig behave like a repeatable RGB film scanner.

The app coordinates the Scanlight, Sony camera, capture folders, calibration, and compositing pipeline. For each film frame it creates three RAW captures under narrowband red, green, and blue light, then composites those captures into a linear DNG and/or TIFF for inversion in Negative Lab Pro, FilmLab, or Lightroom workflows.

Scanlight control Set white, red, green, blue, levels, settle timing, and all-off from the Swift app.
Sony Wi-Fi SDK Connection test, Access Auth fields, still capture, host-PC ARW download, shutter speed control, and live view preview.
Calibration Solves per-channel shutter and LED levels against selected film base, then captures flat-field frames.
Not automatic No motorized film advance, no lens focus/aperture control, and no finished positive conversion inside the app.

NORMAL OPERATOR FLOW

From cold start to scanned frame.

The app is a control hub. It does not replace the physical holder, camera setup, or negative conversion tool. It makes the capture and calibration steps repeatable.

Step What you do What the app does
1. Configure Choose output folder, trigger mode, Sony auth if SDK mode, and composite format. Persists settings locally and validates required fields for the selected mode.
2. Connect Connect Scanlight. In SDK mode, run Check Connection and open Live View if needed. Claims the Scanlight serial port and probes the Sony camera without firing the shutter.
3. Calibrate Select film base/rebate region, keep the lens fixed, remove film for flat field when asked. Captures calibration frames and solves R/G/B exposure settings from real ARWs.
4. Scan Click capture for each film frame, then manually advance the film holder after success. Cycles R/G/B, triggers or waits for captures, names RAWs, and optionally streams composites.
5. Invert Import the linear DNG/TIFF composites into Lightroom, NLP, or FilmLab. Does not perform final negative inversion yet. The output is made for external inversion tools.

What happens during one SDK frame

Red Set red LED level Set solved shutter Trigger Sony capture *_R.ARW
Green Set green LED level Set solved shutter Trigger Sony capture *_G.ARW
Blue Set blue LED level Set solved shutter Trigger Sony capture *_B.ARW

The important invariant: all three captures must be the same film frame. The app only advances its frame counter after the full triplet succeeds; you advance the film manually after that.

Composite output

Streaming composite means the app starts building the linear RGB file in the background after each successful R/G/B triplet. The main useful formats are:

Linear DNG

Best Lightroom-style workflow because it behaves more like a RAW file in the Develop module. Use this for the normal NLP path.

16-bit TIFF

Useful as a stable interchange format and for tools that prefer TIFF. It is already demosaiced and composited.

CAPTURE MODES

Pick the mode that matches the hardware that is actually working.

The modes are interchangeable from the app, but they do not all control the same parts of the rig. SDK is most integrated. Manual / IED is the fallback. Pulse is only for a proven shutter cable.

CALIBRATION AND FFC

Calibration should measure real ARWs, not the preview image.

Live view is for aiming and selecting the film base/rebate region. The color and exposure solve should come from the captured ARW triplet because that is the data used by the final scan.

Recommended calibration behavior

  1. Turn the Scanlight to white for the preview picker so the film edge/rebate is easy to see.
  2. Select the film base or rebate region in live view.
  3. Capture a real dark frame plus R/G/B ARW triplet for that selected region.
  4. Measure the same mapped region in the ARWs and solve per-channel shutter plus LED brightness.
  5. Turn the light off when the sequence ends or fails.
  6. Remove film, then capture flat-field frames with the solved settings.

Exposure solve

The intended target is a high but unclipped film-base signal. In SDK mode the solver uses shutter speed as the coarse control and LED level as the fine trim.

Assumptions: ISO 100, f/8, manual focus lens, fixed camera position, fixed holder position.

Flat field correction

FFC is a no-film capture that records illumination unevenness for each channel. It corrects vignetting and color falloff before the final composite is written.

Re-run it when lens, distance, holder, light position, or film stock setup changes.

Current rough edge: the live-view ROI to ARW crop mapping still needs real-world validation. If the selected strip looks wrong in results, use auto-detect or reselect a clearer rebate area before trusting the calibration.

SONY SDK DETAILS

The SDK path is Wi-Fi PC Remote, not USB.

The current Sony integration uses the Camera Remote SDK over the camera network connection. Access Authentication is required on newer Sony bodies, and the camera can provide both still capture and a live-view JPEG stream.

Connection test

Use Settings -> SDK -> Check Connection before calibration or scanning. It connects, completes auth, then disconnects without firing the shutter.

phase1/sony-capture/build/sony-capture \
  --connect-only \
  --ip-address 10.0.0.247 \
  --username USER \
  --password PASSWORD

Live view

Live view is a sequence of JPEG frames from the camera through the SDK. It is not RAW and it is not the final color measurement path.

phase1/sony-capture/build/sony-capture \
  --live-view-stream-out /tmp/sony-live-view.jpg \
  --ip-address 10.0.0.247 \
  --username USER \
  --password PASSWORD

What live view is and is not

Question Answer
Is it video? Operationally yes, but technically the app displays refreshed SDK live-view JPEG frames.
Is it decoded on the camera? The camera/SDK provides compressed live-view image data. The Mac decodes each JPEG for display.
Is it used for final color? No. Use it for framing and ROI selection. Calibration and scanning should measure captured ARWs.
Does opening live view take a photo? No. It keeps one SDK session open and refreshes a preview frame until you close it.

Auth storage note: the app may store Sony auth locally for convenience. That belongs in local macOS settings or ignored cache files only. Do not commit camera usernames, passwords, or fingerprint blobs to GitHub.

WHAT IS MISSING OR STILL ROUGH

Known gaps are part of the operating procedure.

The rig is useful, but it is not a fully automatic lab scanner. These are the current boundaries to keep in mind while testing.

Area Current state Operator impact
Film advance No motorized advance. You advance the holder by hand after each successful triplet.
Sony USB SDK Deferred for this rig. Use Wi-Fi PC Remote for SDK capture, or Manual / IED as fallback.
RemoteTransfer pull mode Not the main path. Use host-PC SDK download. FTP/NAS can still be useful as a separate backup path.
Live-view to ARW ROI mapping Implemented conceptually, needs real-world validation. Trust calibration results only after checking that the selected rebate maps correctly.
Lens control No autofocus or aperture control. Set focus manually and keep the lens at f/8 before calibration and scanning.
Final inversion Not built into the app. Use Lightroom, Negative Lab Pro, FilmLab, or another inversion workflow on the composites.
Error messages Improving, but some backend errors still surface raw details. Use the failure assistant below to translate common cases.
